System Architecture

The ESP32-S3 CAM AI eBook Reader is architected around multi-core firmware tasks, shared peripherals, and an intuitive LVGL interface. This page explains how hardware and software components interact.

System Block Diagram

FreeRTOS Task Flow

The system splits responsibilities across Core 0 and Core 1: Core 0 handles networking, audio playback/recording, and OTA; Core 1 handles LVGL UI rendering, input, and display tasks.

Data Flow

Book content is read from SD, rendered on the display, and optionally passed to AI or audio subsystems. OCR captures camera frames, converts them to text, and feeds them into note and AI pipelines.

Core Responsibilities

  • Core 0: Network communications, cloud AI requests, audio engine tasks, SD writes, OTA.
  • Core 1: LVGL UI task, touch input, display refresh, user interaction, and rendering.
